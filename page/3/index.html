关于学校校园网挂VPN解决断网问题的个人解读

引言

从上个学期开始，学校推展网络一体化，三家运营商的网络归入CUMT_STU，师生通过身份认证上网。

一体化之后，宿舍里三家运营商的wifi合并到一起，插网线也只需要往同一排插口接就可以了

而且全校基本实现wifi覆盖，特别宿舍的wifi质量非常好（有爽到）

​

不过一体化之后，学校突然限制了同学的上网时段（工作日晚十一点半后禁网）

不少深夜奋战的同学只能选择手机开热点

气抖冷，青年何时才能站起来？（不是）

​

正值众同学半夜断网大喊卧槽之际

有同学发现

在断网时段之前，如果连接校园网，再挂学校的vpn

居然可以实现 在禁网时段，照常使用学校的网络上网！

​

最近刚学了计算机网络，觉得这个操作应该可以用课本的知识解释。

于是便有了这篇博客。

​

要解释这个问题，本文打算先说三个基础知识：

终端如何访问外网、学校的网络情况、vpn的原理

然后再结合以上三点，具体说明校园网挂vpn是如何无视时段禁网的。

​

下面是正文

​

​

三个基本的原理

​

原理一：终端通过路由器访问外网

我们常用的家庭网络都是通过双绞线或者wifi，将手机、pc和ipad等终端连接到路由器，才能上网的。

也就是说，如果要上外网，必须要有路由器！

而通过一个一个的路由器，我们最后就可以访问到某个地点放置的服务器！

​

​

原理二：学校的CUMT_STU使用多个路由器供学生上网

在校园内，虽然会有多个同学都使用CUMT_STU上网

但根据认证页面选择的运营商的不同，我们可以选择不同的路由器上网

​

下面给出上述结论的依据

​

以下是分别使用，校园网和联通，访问外网的经过的ip地址：

使用校园网访问百度经过的路由器（ip地址），可以看到校园网路由器使用的ip地址为 202.119.203.65

​

使用联通访问百度经过的路由器（ip地址），可以看到联通的路由器使用的ip地址为 112.85.229.157

​

可以看到，由于选择的两个运营商不同，访问外网使用的ip地址/路由器也完全不同。

另外，ip地址是对应到路由器的，注意这点后面我们会用到。

​

简单来说，就是在学校，只要是通过wifi或者网线，连接到CUMT_STU的同学

根据他们的选择，一共可以连接到4个路由器：

移动的、电信的、联通的、校园网的，用于访问外网。

​

然后，到了禁网的时段，路由器就不会帮我们向外部发消息，请求服务，所以我们就没法上网了。

因为在禁网时段，我们没办法进行网络认证，路由器就不会帮我们给“外部”发信息。（注意这个外部是伏笔）

（这个认证，每隔一段时间都要进行一次，不单单是首次登录就完事的了）

​

​

原理三：虚拟专用网VPN的基本原理

在原理一我们提到：如果要上外网，必须要有路由器。

但是路由器只是上外网的必要条件，如果要访问外部网站还有其他条件，比如访问权限。

​

如果所有人都可以随便访问外网的每一个服务器，那互联网大企供内部人员访问的数据库，估计没几分钟就给各路人马给白嫖完了…

​

所以，当我们访问外部的主机的时候，目标路由器还会判断我们是否有资格访问。

​

当然，比如百度、微博这些网站是不需要特殊权限的，村里通网了就可以访问。

​

​

那我们如何获得访问权限？

  1. 专门连一条专用的网线过去，其他人不能用（成本高而且浪费资源); ×
  2. 使用虚拟专用网VPN！（终于到重点了！）√

​

vpn的原理很简单，就是把终端发送的信息进行加密，

而只有目标终端那边的路由器才能解密

中间路过的所有路由器都无法解密，（为何中间会有路由器出现，具体见原理一的加粗字）

因此，中间路由器无法截取发送的信息，保证信息中途不会泄露。（只能当纯工具人）

​

另外，通过vpn，不管我们访问哪个ip地址，我们都会把所有的网络请求都先发给目标路由器！

​

这点非常重要，可以理解为：

通过vpn我们不仅有了访问其他组织内网的通道，我们还获取了连接其他外网的新的路径！

以上就是我们要讲的三个原理，下面就是重点的分析了

​

​

​

具体分析

​

上面说了一大堆，那到底为什么校园网挂vpn可以无视时段禁网？？？

​

现在咱们就来分析一下

​ 先来看两个结论：
​ 首先，根据原理二，我们通过校园网认证页面可以选择不同的运营商，选择不同的路由器，访问外网；
​ 其次，根据原理三，我们挂vpn的时候，会把上网的所有请求，从自己的路由器发送到学校的路由器；

​

下面就是重点了！

​

我们来看一下，联通挂vpn和我们使用校园网，分别访问百度的截图：

联通挂vpn上网，离开本地的第一个ip地址【2】，即vpn访问的目标路由器地址，为202.119.203.1

​

使用校园网上网，离开本地的第一个ip地址【3】，即用户使用的路由器地址，为202.119.203.65

​

有没有发现什么？

校园网和vpn链接过去的ip地址前面的网络号是一样的！

​

也就是说，校园网用的路由器 和 vpn的目标路由器 是同一个！！

我们使用校园网和vpn就是通过”同一个路由器”的”不同子网ip”对外网进行访问！

​

有的同学可能就会问：“那有怎么样？到点了不还是禁网吗？”

​

然而，根据原理二，在禁网时段，路由器不会帮我们给“外部”发信息，但是我们仍然可以发信息给路由器。

​

信息传递给路由器之后，由于我们挂了vpn

校园网路由器就会给目标路由器发认证消息

​

而vpn的目标路由器和我们使用的路由器又恰巧是同一个

所以学校的路由器会自己把认证消息发给自己看，不需要访问外部的路由器，所以直接无视断网时间段！

​

最后，我们肯定是可以通过vpn的认证的

​

于是我们就从子网：202.119.203.65（校园网的ip），跳到了子网：202.119.203.1（学校教学用的ip）

​

注意这里的两个子网是同一个路由器里的ip地址

而这个过程是无视断网时段的

​

路由器当然不会在禁网时段内，允许我们使用202.119.203.65访问外网

毕竟我们的校园认证在该时段会失效

​

可是，通过vpn，我们就可以使用202.119.203.1访问外网。

​

而202.119.203.1是一般情况是不会断网的

否则整个学校教学使用的网络都会断掉！

所以这条路线是非常稳定的。

​

所以，根据断网前不同的操作，在禁网的时段，我们访问外网（比如说上B站），就经历如下流程：

①不挂vpn，随便选一个运营商的路由器（包括校园网、移动、联通、电信）：

机子:”我要上B站！” –>

运营商的路由器:”我不认识你，凭什么帮你发消息？” –>

机子:”上不了网了，该歇了~”

​

②挂校园vpn，但是选移动、联通、电信的路由器：
机子:”歪？（打电话给运营商的路由器）帮我找学校的路由器，我要上B站！” –>

运营商的路由器:”我不认识你，凭什么帮你转接？” –>

机子:”上不了网了，该歇了~”

​

③挂别的vpn，用校园网：
机子:”歪？（打电话给校园网的路由器）帮我找学校的路由器！我要上B站！” –>

校园网的路由器:”我不认识你，凭什么帮你转接？” （连不到vpn的目标路由器）–>

机子:”上不了网了，该歇了~”

​

④挂校园vpn，用校园网：
机子:”歪？帮我找学校的路由器！我要上B站！” –>

校园网的路由器:”我不认识你，凭什么……” –>

校园网的路由器:”等等，学校路由器不就是我吗？你等下，我看下你是不是会员哈” –>

“vpn认证成功 “–>

校园网的路由器:”尊贵的客户…您向B站的请求已发出~ “ –>

机子:”收到B站返回的信息，正在生成网页” –>

我:”up主更新啦！爷青回！不歇了！”

​

看这里你可能会担心vpn上网的速度可能不能满足普通的上网需求

​

其实由于此时vpn链接的两个路由器其实是同一个路由器，不用绕远路找目标路由器

这样就能够省去的大量的中间时间，因此速度还是可以的。

​

最后发一下正常连联通，和挂vpn的校园网速度对比：

使用联通上网用speedtest测试，速度还是非常可观的

​

使用校园网挂vpn好吧……不过，这个速度其实还是够用的了

​

​

结语

自此，我尽我所能，大概讲了一下我的理解。由于基本功不扎实，上面的说法应该有不少问题，希望大佬及时指正！

如有错误，还请各位读者原谅……

使用Heroku部署网页

最近学习了使用python的Django框架编写一个笔记条目记录网站。根据参考书的指导，把项目托管到了Heroku上。

下面记录一下，如何把pythonWeb项目托管到Heroku上运行。


1.安装必要的包

如：
dj-database-url
dj-static
static3
gunicorn


2.创建记录包列表的文件 requirements.txt

$ pip freeze > requirements.txt


记得加入 psycopg2>=<2.6.1>

// <>表示不一定是版本号

ps: windows系统好像不太一样


3.在 runtime.txt 指定 python 的版本

  • python –version 查看版本
  • 写进 runtime, 注意用小写

4.修改 settings.py 保证能够在 heroku 的服务器跑起来

可以参照一下我在 learning_log 项目写的设置：

#Heroku设置
if os.environ['HOME'] == '/app':
    import dj_database_url
    DATABASES = {
        'default': dj_database_url.config(default='postgres://localhost')
    }

    #让request.is_secure()承认X_Forward_Proto头
    SECURE_PROXY_SSL_HEADER = ('HTTP_X_FORWARDED_PROTO', 'https')

    #支持所有的主机头(host header)
    ALLOWED_HOSTS = ['*']

    #静态资产配置
    PROJECT_ROOT = os.path.dirname(os.path.abspath(__file__))
    STATIC_URL = 'static/'
    STATIC_ROOT = os.path.join(PROJECT_ROOT, 'static')

5.创建启动进程的 Procfile

不指定扩展名，P为大写

web: gunicorn learning_log.wsgi --log-file -
  • 指定服务器为 gunicorn，
  • 并且使用 wsgi.py 中的设置，
  • 告诉 Heroku 什么类型的事件写进日志

6.创建静态文件目录

1.在 <learning_log/learning_log/> 中创建文件夹 static，
2.并且用 placeholder.txt 占位 // <>表示不一定为这个路径
3.txt 的内容随便写


7.使用 git （没有的就先看怎么安装，这里不赘述）

编写 .gitignore文件，内容如下

11_env/
pycache/
*.sqlite3
// 内容不唯一

再一顿基操：

git init
git add .
git commit -am “”
git status
// 如果 app 已经创建好了
git push heroku master

8.推送到 Heroku

代码如下：

heroku login
// 登录之后
heroku create
git push heroku master

9.建立数据库

code:

heroku run python manage.py migrate

10.有时候直接用 heroku 命令会报错，可以加 –remote heroku 解决


11.改进 Heroku 部署

创建超级用户

code:

heroku run bash
ls
python manage.py creaetesuperuser
balabala...
exit

优化 URL

heroku apps:rename <mps-learning-log> // <>表示不唯一

12.确保项目安全

在setting.py中关闭调试模式，只允许 Heroku 托管

#只允许 Heroku 托管这个项目
ALLOWED_HOSTS = ['*']
DABUG = False

推送并提交修改

《算法图解》内容概要

浅谈一下《算法图解》

打算系统的学一下算法，上个月刚看完《算法图解》，这周刚好抽出时间，来写一写该书的总结。

本文打算先大致谈谈本书的特点和大概内容，对于个人比较感兴趣的地方再稍微做展开，最后谈谈本书给我的感受

本书特点

作为一本面向完全初学者的算法书，本书旨在，用最简单的表达，为读者介绍算法学习中常见的几个概念，并且介绍了几个比较基础的算法，最后再简要提及了几个算法，引导读者进一步学习。

大致内容

全书分为11章，脉络清晰。

  • 第一章介绍了一下大O表示法和二分查找；
  • 第二章开始解释选择排序，并且解释了一下内存的工作原理，以及数组和链表的概念；
  • 第三章开始介绍的简洁优雅的基础算法—— 递归；
  • 第四章用快速排序算法的例子，介绍了一下 分而治之 的算法思想；
  • 第五章主要介绍了散列表的算法实现（通讯录）；
  • 第六章介绍了广度优先搜索算法，顺便解释了“图”的概念；
  • 第七章趁热打铁，介绍了狄克斯特拉算法，作为广度优先搜索的进阶算法；
  • 第八章又介绍了另外一种耳熟能详的算法——贪婪算法；
  • 第九章以背包问题的优化入手，介绍了动态规划的算法思想；
  • 第十章介绍了k最邻算法，这个算法是很多推荐系统和机器学习的基础算法；
  • 第十一章的内容比较杂，简单介绍了很多进阶的算法概念，为读者提供了进一步学习的方向。

下面贴上我的思维导图

该图作为参考脉络，对于有兴趣的概念各位可以逐一去了解。

下面来聊一聊几个有趣的点

大O表示法

大O表示法是一种描述算法运行速度的方法。它指出了最坏的情况下算法运算量与数据量之间的关系。我们一般通过它来比较不同算法的运行效率。常见表示为O(p(n))

ps:所以为什么是用“O”？

举个例子：

如果我要在一张纸上画四个方格，最笨的方法是在纸上一个格子一个格子的画，比较好的方法是将整张纸对折两次。

于是前者的大O表示为O(n)；后者的大O表示为O(log n)

注意
  • “n”表示数据量，比如本问的n=4，O(n)表示第一种方法的运行时间为n
  • “log n”表示底数为2的对数函数，比如本问第二种方法的运算时间为”log 4” = “2” （很多算法问题用到的底数都是2）
  • 大O表示法不考虑常数，比如说问题变成“在两张纸上各画四个方格”那么上述两种方法的大O表示法不变

显然，O(log n)要比O(n)简单地多。实际上，O(log n)已经是运算量非常小的算法了。

常见的大O表示法

这样看来大O表示法的种类有很多，但是常见的大O表示法其实并不太多，下面大致罗列一下。

  • O(log n) 对数时间，比如二分查找就是这种算法
  • O(n) 线性时间
  • O(n * log n) 这样的算法包括快速排序，后面会讲到
  • O(n ^ 2) 包括选择排序，后面也会讲到
  • O(n!) n的阶乘，一种非常慢的算法，但是对于NP问题无可奈何的时候也可以采用的方法，但也可以用贪婪算法等近似求解代替正解
关于大O表示法还有很多概念这里不一一展开，希望感兴趣的同学自行了解。

聊聊排序

给定一个无序的数组，怎样才能得到一个排好序的数组呢？这是很常见的一个问题。
这个问题看似简单但是处理起来很麻烦，因为我们必须逐个逐个元素进行确认。

选择排序

比较简单的一个方法是选择排序。它的原理如下：

比如数组为{2, 6, 8, 9, 5, 7}，需要从大到小进行排序。可以对数组中的每个元素进行比较，得到最大值，放到新数组的首位；后面对剩下的元素重复上诉操作，把得到的最大值放在新数组的后一位。于是我们就得到了拍过序的新数组{9, 8, 7, 5, 6, 2}。

这种排序方法就是选择排序

由于这种算法需要找n次最值，每次找最值需要比较n次，所以它的运行时间为O(n ^ 2)。
那么问题来了：各位同学可能觉得这个结果不太对劲。因为找完一次最值之后，下一次只需比较n-1次就可以了，所以运行时间应该为O(n + (n-1) + (n-2) + ... + 1)才对
而(n + (n-1) + (n-2) + ... + 1)是有个等差数列，等效于(n * (n+1) / 2)，由于大O表示法无视常数，所以最后的运算时间则为O(n * n)
看到这里，你可能会觉得大O表示法不科学，因为它无法准确表示运算时间，但是大O表示法旨在表示算法复杂度，不够抽象反而有失偏颇。
另外，大O表示法表示算法最遭的运行时间，而实际计算时间的时候会用“平均运行时间”来表示，有兴趣的同学可以了解一下。
